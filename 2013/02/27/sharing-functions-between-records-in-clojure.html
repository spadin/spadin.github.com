Sharing Functions Between Records in Clojure

Today I came across a nice way to share functionality between records that extend the same protocol in Clojure. I recently wanted a way to encapsulate some functionality into a record via defrecord but noticed some duplication, after a little digging around I found some nice tips for doing this. Unfortunately, I canâ€™t find those links in my history. I have the next best thing though, some test code examples.

Notice in the example below that both TextPrinter and HtmlPrinter have access to the ouput function and upper-case function. HtmlPrinter overrides the default output function while TextPrinter keeps both defaults. This pattern may work well if you would like to reduce duplication and separate these records into difference namespaces.

Sample code:

    ; src/protocol_test/core.clj
    (ns protocol-tests.core)

    (defprotocol IPrinter
      (output [this message])
      (upper-case [this message]))

    (def PrinterDefaults
      {:output (fn [this message]
                    (print message))
       :upper-case (fn [this message]
                     (clojure.string/upper-case message))})

    (defrecord TextPrinter [])
    (defrecord HtmlPrinter [])

    (extend TextPrinter
      IPrinter
      (merge PrinterDefaults))

    (extend HtmlPrinter
      IPrinter
      (merge PrinterDefaults
             {:output (fn [this message]
                           (print (str "<p>" (upper-case this message) "</p>")))}))


And the tests:

    ; spec/protocol_tests/core_spec.clj
    (ns protocol-tests.core-spec
      (:use [speclj.core]
            [protocol-tests.core])
      (:import [protocol_tests.core TextPrinter HtmlPrinter]))

    (describe "core"
      (it "returns the default functionality"
        (should= "sample text"
                 (with-out-str (output (TextPrinter.) "sample text"))))

      (it "returns the SAMPLE TEXT surrounded by a p tag"
        (should= "<p>SAMPLE TEXT</p>"
                 (with-out-str (output (HtmlPrinter.) "sample text")))))


Aside:

Notice how I had to import the records in the spec. I had to use an underscore in protocol_tests.core which I believe is a bug in the current version of Clojure (1.4).
