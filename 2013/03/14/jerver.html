Jerver, part 1

A Java-based web server

I’ve spent this week learning about sockets and reading RFCs in order to create a Java-based web server. I call it Jerver. (Can you tell I like portmanteaus; Ticlj & Taclj) It’s been a fun and challenging week.

I was pretty worried when I was tasked to create a web server in one week. Granted, the requirements did not call for a fully-featured web sever, but a web server, nonetheless.

While I’ve never written a web server, I have researched the topic a bit for my own knowledge.

My initial accessment was that a server would need to:

    * Accept a connection
    * Parse a request
    * Interpret headers
    * Parse HTTP params
      * Parse content-length
    * Determine HTTP method (GET, POST, HEAD, etc)
      * Route a request to a resource
      * Determine the mime type of the resource
      * Write the response back through the connection
      * Close the connection

While there are more intermediary steps that I may need to add eventually, these simple steps are enough to get a web-server running.

I will be writing a series of posts on my design decisions for implementing this web server. As mentioned above, this is not a feature-complete web-server, but could serve as a good way to learn about servers and a good starting point for your own implementations.
