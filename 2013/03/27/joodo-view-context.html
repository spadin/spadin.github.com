Joodo *view-context*

Joodo is a web framework for Clojure, similar to Sinatra for Ruby.

If you’ve tried Sinatra you’ve probably used variables inside of views. A common way to do this is by setting an instance variable in the controller:

    get '/hello/:name' do
      @name = params[:name]
      erb :hello
    end

which then magically grants you the ability to use that variable in the view:

    <p>Hello <%= @name %></p>

I say magically, because it’s not clear up front how those instance variables get passed to the view. Hint: Sinatra, like Rails, uses Tilt to render templates, which allows setting a scope for the template that is being rendered.

Enough about Ruby, what does this have to do with Joodo and Clojure?

Joodo has an analogous way of passing data to your Hiccup view; it’s done via the *view-context* binding.

Rendering a Hiccup template with using is done in a controller via:

    (GET '/hello/:first-name' [] (fn [first-name]
      (render_template 'hello' :first-name first-name)))

Now in your hello.hiccup.clj you are able to retrieve and use the first name as such:

    [:p (str "Hello " (first-name *view-context*))]

or nicer yet, especially if you use first name more than once.

    (let [first-name (:first-name *view-context*)]
      [:p (str "Hello " first-name)])

Conclusion

Passing parameters to Hiccup views in Clojure is more akin to passing locals with Sinatra in that they have to be explicitly declare. I didn’t lead with that though since a majority of people are familiar with sending and using the instance variables instead.

Final note: The Joodo *view-context* is a globally available binding for the current request that is being rendered. As such, it can be accessed from your layout.hiccup.clj as well as any partials that you may need to render as well.
